# Claim Taxonomy

Complete parameter schemas for all 14 built-in claim types. These describe what the extraction prompt produces and what the verifier functions consume.

## File Claims

### FILE_EXISTS

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `path` | string | yes | Relative file path from repo root |

**Extraction produces:** `{"path": "src/config.py"}`

**Verifier checks:** `os.path.isfile(repo_path / path)` after `safe_path()` traversal check.

---

### LINE_CONTENT

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `path` | string | yes | Relative file path |
| `line` | integer | no | Expected line number (1-based, approximate) |
| `content` | string | yes | Expected content substring |

**Extraction produces:** `{"path": "model.py", "line": 42, "content": "torch.load"}`

**Verifier checks:** reads file, gets line at `line` number, checks if `content` is a substring of that line.

!!! note
    The schema uses `content` for the expected text, but the verifier code reads the parameter as `expected`. Both names work, but `content` is the canonical key in the extraction schema.

---

### FILE_CLASSIFICATION

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `path` | string | yes | Relative file path |
| `classification` | string | yes | Expected classification: `test`, `vendored`/`vendor`, or `production` |

**Extraction produces:** `{"path": "tests/test_auth.py", "classification": "test"}`

**Verifier checks:** path against regex patterns. Test patterns: `test[s_/]`, `_test.`, `.test.`, `spec[s_/]`, `e2e/`, `integration/`, `fixtures?/`. Vendor patterns: `vendor/`, `third_party/`, `node_modules/`. Everything else is `production`.

---

### GENERATED_OR_VENDORED

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `path` | string | yes | Relative file path |
| `expected` | boolean | no (default: true) | True if expected to be generated/vendored |

**Extraction produces:** `{"path": "api/generated.pb.go", "expected": true}`

**Verifier checks:** vendored path patterns, then reads first 1000 bytes looking for markers: `DO NOT EDIT`, `AUTO-GENERATED`, `GENERATED BY`, `Code generated by`, `@generated`, `auto-generated`, `THIS FILE IS GENERATED`.

---

## Function Claims

### FUNCTION_EXISTS

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `name` | string | yes | Function or method name |
| `file` | string | no | File where it should exist |

**Extraction produces:** `{"name": "parseConfig", "file": "config.go"}`

**Verifier checks:** grep for language-specific definition pattern. Language is detected from `file` extension if provided, otherwise from the finding file.

**Patterns by language:**

| Language | Pattern |
|----------|---------|
| Python | `def {name}\s*\(` |
| Go | `func\s+(?:\([^)]*\)\s+)?{name}\s*\(` |
| TypeScript/JS | `(?:function\s+{name}\|(?:const\|let\|var)\s+{name}\s*=)` |
| Java | `(?:public\|private\|protected\|static\|\s)+[\w<>\[\]]+\s+{name}\s*\(` |
| Rust | `fn\s+{name}\s*[<(]` |
| C/C++ | `[\w*:]+\s+{name}\s*\(` |
| Unknown | `(?:def\|func\|function\|fn)\s+{name}\s*\(\|{name}\s*[=:]\s*(?:function\|\()` |

---

### FUNCTION_CALLED

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `name` | string | yes | Function name |
| `expected` | boolean | no (default: true) | True if expected to be called, false if expected NOT to be called |

**Extraction produces:** `{"name": "torch.load", "expected": true}`

**Verifier checks:** grep for `{name}\s*\(`, then subtracts lines matching the function definition pattern. Remaining lines are call sites. Compares `found == expected`.

---

### HAS_CALLERS

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `name` | string | yes | Function name |
| `expected` | boolean | no (default: true) | True if expected to have callers |

**Extraction produces:** `{"name": "handleRequest", "expected": true}`

**Verifier checks:** same mechanism as FUNCTION_CALLED. Greps for call pattern, subtracts definitions, checks if any call sites remain.

---

## Dependency Claims

### IMPORT_EXISTS

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `module` | string | yes | Module or package name |
| `file` | string | no | File where import should exist |

**Extraction produces:** `{"module": "os.path", "file": "src/utils.py"}`

**Verifier checks:** grep for language-specific import patterns.

**Patterns by language:**

| Language | Patterns |
|----------|----------|
| Python | `import\s+{module}`, `from\s+{module}\s+import` |
| Go | `"\s*{module}\s*"` |
| TypeScript/JS | `import\s+.*from\s+['"].*{module}`, `require\(['"].*{module}` |
| Java | `import\s+.*{module}` |
| C/C++ | `#include\s+[<"].*{module}` |

---

### PACKAGE_VERSION

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `package` | string | yes | Package name |
| `version` | string | no | Expected version or constraint |

**Extraction produces:** `{"package": "lodash", "version": "4.17.15"}`

**Verifier checks:** searches lockfiles in order: `go.sum`, `go.mod`, `requirements.txt`, `poetry.lock`, `package-lock.json`. Each has a dedicated parser. Version match is exact or substring.

---

### DEPENDENCY_TYPE

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `package` | string | yes | Package name |
| `dep_type` | string | yes | Expected type: `direct` or `transitive` |

**Extraction produces:** `{"package": "PyYAML", "dep_type": "direct"}`

**Verifier checks:** searches manifest files (`go.mod`, `requirements.txt`, `package.json`, `Pipfile`, `pyproject.toml`). If the package name appears in a manifest, it's classified as direct. Otherwise, transitive.

---

### CVE_AFFECTS_VERSION

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `cve` | string | no | CVE identifier (e.g., CVE-2024-1234) |
| `package` | string | yes | Package name |
| `version` | string | no | Installed version |

**Extraction produces:** `{"cve": "CVE-2024-1234", "package": "PyYAML", "version": "5.3.1"}`

**Verifier returns:** always UNVERIFIABLE. CVE version range checking requires an external advisory database.

---

## Code Claims

### ABSENCE

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `pattern` | string | yes | Pattern that should be absent |
| `scope` | string | no (default: `repo`) | Scope: `file`, `directory`, or `repo` |
| `file` | string | conditional | File path (required when scope=file) |
| `directory` | string | conditional | Directory path (required when scope=directory) |

**Extraction produces:** `{"pattern": "eval(", "scope": "file", "file": "src/handler.py"}`

**Verifier checks:** `grep -rn -F {pattern} {scope_path}`. VERIFIED if zero matches (pattern is absent). REFUTED if any matches found.

---

### MITIGATION_EXISTS

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `mitigation` | string | yes | Description of the mitigation |
| `file` | string | yes | File where mitigation should exist |
| `pattern` | string | no | Code pattern to search for |
| `line` | integer | no | Line number where mitigation is expected |

**Extraction produces:** `{"mitigation": "input sanitization", "file": "handler.go", "line": 58}`

**Verifier checks:** reads the file, checks that the specified line exists and contains non-trivial content (> 5 characters). This is a presence heuristic, not a semantic check.

---

### ENTRY_POINT

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `name` | string | yes | Function or endpoint name |
| `type` | string | no | Entry point type: `http`, `rest`, `api`, `grpc`, `rpc`, `cli`, `command` |

**Extraction produces:** `{"name": "/api/users", "type": "http"}`

**Verifier checks:** grep for framework-specific patterns based on type. If no type specified, searches all pattern categories (HTTP + gRPC + CLI).

**Patterns searched:**

| Type | Patterns |
|------|----------|
| http/rest/api | `HandleFunc`, `.GET(`, `.POST(`, `.PUT(`, `.DELETE(`, `@app.route`, `@router.`, `.HandleFunc(`, `http.Handle` |
| grpc/rpc | `Register.*Server`, `pb\..*Server` |
| cli/command | `cobra.Command`, `argparse.`, `click.command`, `flag.` |
